函数传实参还是形参问题

在 Python 中，参数传递始终是 引用传递（传地址的）。但根据对象类型（可变/不可变）的不同，会产生类似「值传递」或「引用传递」的表象：

结论：看对象类型

  1. 不可变对象（字符串、数字、元组等）作为参数时：
    • 形参接收到的是对象的引用
    • 若在函数内修改形参，会创建新对象，实参不受影响（类似值传递）
    • nonlocal声明实现访问外部变量？
  2. 可变对象（列表、字典、集合等）作为参数时：
    • 形参接收到的也是对象的引用
    • 若在函数内 原地修改 形参（如 append、+= 等操作），实参会同步变化（类似引用传递）

class Solution:
    def zigzagLevelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root: return []
        res, queue = [], collections.deque()
        queue.append(root)
        while queue:
            tmp = []
            for _ in range(len(queue)):
                node = queue.popleft()
                tmp.append(node.val)
                if node.left: queue.append(node.left)
                if node.right: queue.append(node.right)
            res.append(tmp[::-1] if len(res) % 2 else tmp)
        return res

func(*args,**kwargs)

函数定义中，可以有一个 *args 和 一个 **kwargs 参数，args 和 kwargs 是约定俗成的参数命名。也就是说可以func(*a,**b)

我的理解：

  • args是一个元组，*args是解包，即 未知元组(a,b…) 解包成 a,b… 放到func的括号里，变成 func(a,b…)
  • kwargs是一个字典未知字典{a: b, c: d…}，解包放func里，变成 func(a=b, c=d…)

使用式理解：

*func(fixed_arg, args, obvious_kwarg=123, **kwargs)中

  1. fixed_arg 传递：它是固定位置参数，没有默认值。
  2. *args 接收所有多余的位置参数：所有未被 fixed_arg 接收的位置参数都会被打包到 *args。
  3. obvious_kwarg 是仅限关键字参数：它只能通过关键字传递，不能通过位置传递。
  4. **kwargs 接收所有多余的关键字参数：所有未被 fixed_arg、*args 和 obvious 接收的关键字参数都会被打包到 **kwargs。

    print(f"位置参数: {fixed_arg}")		 # 10
    print(f"多余的位置参数: {args}")	   # (20, 30)
    print(f"显式传递的关键字参数: {obvious_kwarg}")	# 60
    print(f"可变关键字参数: {kwargs}")	   # {'x': 40, 'y': 50}

async def _handle_entity_relation_summary(
    entity_or_relation_name: str,
    description: str,
    global_config: dict,
) -> str:
    use_llm_func: callable = global_config["llm_model_func"]
    llm_max_tokens = global_config["llm_model_max_token_size"]
    tiktoken_model_name = global_config["tiktoken_model_name"]
    summary_max_tokens = global_config["entity_summary_to_max_tokens"]
    language = global_config["addon_params"].get(
        "language", PROMPTS["DEFAULT_LANGUAGE"]
    )

    # description=dsri_1<SEP>dsri_2<SEP>...
    # 获取描述的tokens并判断是否要总结
    tokens = encode_string_by_tiktoken(description, model_name=tiktoken_model_name)
    if len(tokens) < summary_max_tokens:  # No need for summary
        return description
    # 将多个dsri 总结成一个综合的
    prompt_template = PROMPTS["summarize_entity_descriptions"]
    # 保证在llm最大tokens限制下
    use_description = decode_tokens_by_tiktoken(
        tokens[:llm_max_tokens], model_name=tiktoken_model_name
    )
    # description_list: dsri_1<SEP>dsri_2<SEP> 被<SEP>分割
    context_base = dict(
        entity_name=entity_or_relation_name,
        description_list=use_description.split(GRAPH_FIELD_SEP),
        language=language,
    )
    use_prompt = prompt_template.format(**context_base)
    logger.debug(f"Trigger summary: {entity_or_relation_name}")
    summary = await use_llm_func(use_prompt, max_tokens=summary_max_tokens)
    return summary
